Introducing Java.

 Objects

Java is an 'object oriented' programming language. The programming language has keywords and a language syntax. You use the language to define types of 'objects'.
Now objects are tricky to describe. At runtime, objects are built in the computers memory. Objects can contain both 'properties' (data variables) and 'behaviors' (processing logic). They can be programmed to do useful work and they can send 'messages' to one another.

Java is also a runtime environment. Java code is interpreted at runtime by the Java Virtual Machine (JVM). The JVM provides a standard set of services to your Java objects, so that they can run using any type of operating system.
The JVM is the central piece of the Java Runtime Environment or JRE. The Java Language Specification defines the programming language and the contract between the JRE and all Java objects. You launch a Java application by running the java executable and passing it the name of your application.

Java relies upon the services of the Java compiler, to validate your Java source code (source files have a .java suffix) and produce the runtime files (which contain instructions for the JVM and have a .class suffix). You compile java source code, by running the javac executable and passing it the name of your source file.

The Java Development Kits (J2SE, J2EE, J2ME), provide libraries of object definitions, which your programs can use to handle data, draw graphics, create windows and buttons, do file and network input and output, check security permissions ...etc All the built-in libraries have names that begin java.something.ClassName or javax.something.ClassName.
The kits also include the Java Runtime Environment (java.exe), the command line tools (including javac.exe), and documentation (as a seperate download). See the links below for more info.

The current version of Java is 1.4. Confusingly, the recent releases (after version 1.1), are all referred to as the 'Java 2 platform'. There are actually several 'flavors' of the development kit, which you can download for free from Suns site.

  • The J2SE standard edition is for writing programs that run locally or in a web browser. It contains a file called src.zip that contains the source code for the public classes for information only.
  • The J2EE is the enterprise edition, and includes libraries of object definitions, for client - server and n-tier programming. This kit contains the libraries that support server side programming (servlets and EJBs).
  • The J2ME contains libraries and a runtime envorinment optimized for PDAs, and 'micro' devices.
