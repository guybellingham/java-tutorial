Events and Listeners

 Example

When registering as an EventListener, the object that adds itself to the list of listeners kept by a component, must implement the appropriate EventListener interface and methods.

For example if your listener object, wants to handle Checkbox ItemEvents, it must implement ItemListener, and define a method with the signature

public void itemStateChanged(ItemEvent e) { }

Only then can it register with the Checkbox as an item listener.

//Register myItemListener as an Itemlistener on myCheckbox
myCheckbox.addItemListener(myItemListener);

The diagram below lists the major event listener interfaces, and some of their more commonly used event handling methods. Because these are interfaces, they define abstract methods. Your event listener must override these methods to do something useful (or nothing at all) in order to avoid becoming abstract also. Java provides Adapter classes (for the listener interfaces with many methods) which your event handlers can extend, and override only those methods they are interested in.
