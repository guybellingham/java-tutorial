SWING

 JComponent

The Swing classes were introduced with Java version 1.1 (as a seperate download in the com.sun.java.swing packages). They were part of an effort by Sun, IBM and Netscape to develop a more mature set of class libraries, collectively called the Java Foundation Classes (JFC). The current version of swing resides in a new set of language extension packages that begin with javax.swing. To view the Swing Applets on these pages, I suggest you visit the Opera website for a FREE, powerful, small and fast browser, that has up to date Java support as standard!

The Swing classes deal with graphical user interfaces, and are built upon the AWT classes. They add a lot of functionality that was 'missing' from the AWT, like support for keyboard input, copy & paste, drag and drop and focus traversal. Swing adds a more sophisticated set of events, and more advanced interface controls like an editor and scrolling panes, a tree view and a table.

Swing has its own versions of the AWT components, with class names that begin with the letter J.

The Swing graphical controls are based upon a different architecture than AWT. They are termed 'lightweight' components, because instead of relying upon the underlying user interface widgets of the operating system (peers), they handle much of the screen drawing and behavior themselves (however this is not true for the Swing containers which still rely upon the underlying operating system 'windows').

To see how powerful the new Swing classes are, try this painting Applet.

Swing makes use of the 'Model-View-Controller' architecture (borrowed from the SmallTalk language). MVC separates the presentation of a component (the 'look and feel') from the underlying data model. In practice the interactions between the 'View' and 'Controller' parts are very complex (they are tightly coupled), so they are combined into a single class called the 'delegate' or UI.
The mother of all Swing user interfaces is the ComponentUI class, from which all others are descended. For example the JButton component has a user interface called ButtonUI class, which extends ComponentUI and is responsible for painting the button shape, icon, label and reporting back the preferred and minimum sizes etc.
