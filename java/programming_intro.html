Programming Java.

 class keywords

Java is an 'object oriented' programming language.
The programming language has keywords and a language syntax, and relies upon an extensive library of built-in runtime classes. When you code with an object oriented language, you must learn to break up your logic and data into self contained functional pieces (objects). The smaller and more generic these pieces the more robust and flexible will be the software you develop.

Designing your solution before you start coding and testing is obviously a good idea, and this is even more important when using objects. A good design will make the development process easier. The symptoms of a good design are an increase in the reuse of existing classes, a seperation of the interfaces (the what) from the implementations (the how), and more flexible (polymorphic) software. Symptoms of a bad object oriented analysis and design, are tight coupling between your classes, large classes, multi-function classes and inflexible (proprietary) solutions.

Java programming syntax and tips.
A Java statement forms a complete unit of execution and is terminated with a semicolon (;). There are three kinds of statements: expression statements, declaration statements, and control flow statements. In control flow statements you should always group your statements together into a block with curly braces. Pairs of curly braces { }, enclose 'blocks' of functionally related statements. They are often used in conditional and program flow statements, where a group of more than one statements, is treated like a single statement. For example :

if (conditional test) {
  // do a group of statements; 
} else { 
  // do some other statements; 
}

Java comments start with /* (or /** for javadoc) and end with */, or follow a double slash // to the end of that line. Comments cannot be nested. The javadoc tool, can build documentation for your classes in html format, by reading your java source files. It looks for comments that start with /** and end */.

As with any programming language, some words are keywords reserved for the operators and verbs of the language itself. The package java.lang.* contains most of operators and methods built into the Java language and is automatically available to any class you write.

Java is case sensitive! This means a Rose is not a ROSE is not a rose! Class, method and variable names can contain numbers but cannot start with them. Names cannot contain spaces, tabs or line feeds and you should avoid unprintable and 'special' characters.

Because a single syntax error, can sometimes have 'knock on' effects, the compiler will sometimes generate a whole bunch of seemingly nonsensical error messages. Try to find the 'real' errors, ignore the others, and they will often go away when you recompile. Common Errors when writing java include the following :

  • Have you mis-spelled a word (remember a Rose is not a rose)?
  • Did you leave any unpaired braces or parentheses?
  • When calling a method, are you forgetting the parentheses after the method name?
  • Did you forget the semicolon at the end of a statement?
  • Did you confuse the assignment operator = (single equals) with the equals comparison == (two equals signs)?
  • Have you used a Java keyword or reserved word as a variable or method name?
  • Did you add a return type to your constructor method by mistake?
  • Are you declaring an object variable but forgetting to assign it to a new object?

Tip! Try not to hard-code values into your programs. Design them so that they inquire about the current properties of components, instead of assuming a fixed value. Try to make the classes you code, as self sufficient and as 'dynamic' and flexible as possible.
