Servlets are 'server side' components, that extend the functionality of the server. In a manner similar to Applets; Servlets are hosted by a Java enabled web server, and run under the control of the Java Virtual Machine. However Servlets have no graphical interface, and work 'behind the scenes'.

All Servlets conform to the Servlet interface, through which they can be invoked by the web server. The most common way to build a Servlet is to extend either the javax.servlet.GenericServlet or javax.servlet.http.HttpServlet classes.

Servlets are a good alternative to the Common Gateway Interface (CGI) for dealing with user input from a web client.
1. Servlets have advantages in performance : they can run within the server process, are multithreaded and compiled into Java byte-code.
2. Servlets remain in memory until explicitly 'destroyed' by the server, and they can share resources such as database connections across multiple requests for service.
3. Servlets are more robust and secure than cgi scripts, because they are written in Java, run within the Java Virtual Machine and can make use of the SecurityManager and fine grain security policies.

Servlets are protocol independent (servlets can extend any kind of server that supports them), and come with built-in support for the HTTP protocol. Servlets are designed to handle Request - Response type applications in a clean and simple manner.

figure: An Http GET request to a Servlet.
 An HTTP GET method request, is commonly used to request 'static' resources or functions from a web server.
The query parameters follow the ? in the URL.
Notice that the parameters are passed as name=value pairs in the URL of a GET request. The URL is specially encoded to remove characters that may be dropped or misinterpreted by the TCP/IP network protocol (spaces are replaced with +, and special characters are expressed as % followed by their ASCII number in hexadecimal base). There are limits on the size of parameters passed in the header of a HTTP GET request. A GET request is often generated by an HTML <A HREF=> link or <FORM>.
figure: An Http POST request to a Servlet. An HTTP POST request is usually used to process dynamic user input, such as a search or login request.
The POST method of the HTTP protocol is more flexible than the GET request. The query parameters are passed in the body of the request, and are not visible in the URL generated by the browser. There is no limit on the number / size of parameters passed in the body of a POST request. A POST request is usually generated by hitting the submit button on a HTML <FORM>, displayed in a web browser.
