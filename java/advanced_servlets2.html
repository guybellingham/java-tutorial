figure: The major servlet classes.
 The servlet classes are packaged as Java Standard extensions (packages beginning with javax), which means they are not a required part of the Java runtime.

These classes are distributed as part of the Java Servlet Development Kit (JSDK).
 
The GenericServlet class, provides a framework upon which you can write Servlets that can process any type of Request - Response protocol such as FTP, POP, Telnet or custom protocols.
The HttpServlet class, contains methods specific to handling HTTP (Hyper Text Transfer Protocol) requests from a web page. Extend this class, to make use of its default properties and behavior, and then override those methods you need, to tailor it to your needs. When writing your own HttpServlet, you will need to import both javax.servlet.* and javax.servlet.http.* packages.
