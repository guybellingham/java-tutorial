The class statement.

 reusing classes

Java defines types of objects with the class statement. A class, is the way you define a new type of object in Java (when you see the word class, think 'type').
In fact, each source file (source files have a .java suffix) should have a single public class statement in it. To help Java find the correct file at runtime, the filename must be the same as the class name.
So for example, if I'm defining a Circle class, my source code must be in a file called Circle.java. And when my code compiles successfully, my runtime code will be in a file called Circle.class.

Class files exist as files or archives on your computer or on an internet server. They are organized in file system folders. We'll talk about how they are packaged on a following page. Class files actually contain 'byte code', which the Java Virtual Machine (JVM) interprets. Class files act as a sort of template, from which objects are built at runtime by the JVM.

Class names start with a Capital letter and are usually a 'real world' thing or noun. Note that class names do not contain spaces and most importantly that all names in Java are case sensitive!
Here is a simple empty class statement. 

// This class defines a Circle
package my.shapes;
public class Circle extends Shape {
   /* member variables and
      member methods go here */
}

Syntax Note:
Curly braces { ... } serve to contain blocks of code in Java.
All statements in Java code must end with a semicolon ;.
The greyed out text are comments. You will find it is important to document your code thoroughly.

A class statement also serves to indicate what existing object type the current class is based upon. Java uses the keyword extends followed by the parent class name, to identify an inheritance relationship. This is what the extends keyword in the above example is doing, it is saying that objects of type Circle inherit from the Shape class. Actually the extends keyword is optional, without it, Java automatically assumes that your class inherits directly from the java.lang.Object class. We'll talk more about inheritance on the next page.

You can list any programming interfaces that your class is built to service, in the class statement after the implements keyword.
We'll talk about programming interfaces on a following page, but for now you need to know that Java classes are allowed to implement multiple interfaces, and by doing so, the class is declaring that it provides runtime behavior for all the methods defined in those interfaces.
For example my Circle class implements the Cloneable interface like this : 

package my.shapes;
public class Circle extends Shape implements Cloneable {
  //properties and methods of a Circle go here 
  float radius; 
  public Object clone() {
  } 
}

In the example above, by implementing the Cloneable interface all objects of the Circle class are stating that they implement the clone() method, which returns another Object of some kind which is an exact copy of themselves.

A class statement contains the member variables (numbers, text, arrays and references to other objects) which embody the runtime properties of that particular object. You also code member methods, which embody the behaviors of that runtime object, inside the class statement.

The Java compiler expects your source code in a particular order:

  • A package statement (if needed) must precede all others in the source file.
  • Next you can list any import statements your class might require. There's more details about packages and the import statement on the following pages.
  • Comments can appear anywhere within your class definition, and are ignored by the compiler.
  • The class statement, is followed by curly braces which surround the member variables and methods that belong to the class.
