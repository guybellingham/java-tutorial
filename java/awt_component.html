java.awt.Component

 Button
The Component  class is an abstract class that represents a generic user interface ‘component’, which can be anything from a window to a button.

All Components can have a 'parent' which is a Container.

All Components have a physical onscreen size, and location.

All Components can notify 'event listeners', when certain kinds of events happen, such as when a key is pressed or the mouse is moved or clicked over them. See the methods described below for the Java 1.1 event model.

In addition Components have graphical attributes such as Colors, Fonts and whether or not they are visible.

In the table below are some descriptions of the methods that all AWT Components inherit. Note that a few of these methods are not public, and are therefore not for use by you. Some others have been 'deprecated' and you should use the newer alternative methods that they refer you to.

method   description

public void addNotify( ) ;

 method that is used by the AWT toolkit to build the ‘peer’ for this Component in the native windowing system, making it displayable by the operating system.

public Rectangle bounds( ) ;
public Rectangle getBounds( );

 both methods return the Rectangle x, y, width and height that encompasses this component. The bounds( ) method has been replaced (or deprecated) by the getBounds( ) method for Java 1.1.
public void setBounds(int x, int y, int width, int height); moves and resizes this component. The new location of the top-left corner is specified by x and y, and the new size is specified by width and height.
Used by LayoutManagers to layout this component.
Replaces the older reshape( ) method.

public void resize(int height, int width ) ;
public void setSize(int height, int width ) ;

 the resize( ) method has been replaced by the setSize( ) method in Java 1.1.
This method attempts to dictate the size of a component. This is usually futile, because a LayoutManager will determine the true display size of a component (an exception is made for the Frame Component).

public Point getLocation(); 
public void setLocation(Point p);

 gets the location of this component in the form of a point specifying the component's top-left corner. The location will be relative to the parent's coordinate space.
attempts to move this component, so that its top, left corner is at the point specified. setLocation( ) replaces the move( ) method for Java 1.1

public boolean contains(int x, int y) ;
public boolean inside(int x, int y) ;
public boolean contains(Point p) ;

 all these methods return true if the coordinates x and y fall within the bounds of the component. Also accepts a Point object as an argument.

public void disable( ); 
public void enable( );

 Deprecated methods, see the newer versions of this method below.
A disabled component does not respond to input, and often paints itself ‘greyed out’.

public void setEnabled(boolean b);

 if the argument b is true, the component is enabled and displays a normal appearance, otherwise it displays 'greyed out'.
Replaces the older Java 1.0 enable( ) and disable( ) methods.

public boolean isEnabled( ); 

 tests to see whether the component is enabled. 

public Color getBackground( ); 
public Color getForeground( );

 returns the background and foreground colors of the component as an object of the java.awt.Color class.

public void setBackground(Color c);
public void setForeground(Color c);

 sets the background and foreground colors of the component. If you do not explicitly set these, the component uses the background and foreground of its immediate container.

public Font getFont( ); 
public void setFont(Font f);

 returns the Font currently used by the component. 
sets the font that the component will use for rendering any text. If you do not explicitly set the font, the component uses the font of its container.

public Graphics getGraphics( );

 returns the Graphics context object for drawing or painting onto this component. 

public String getName( );

 gets the ‘name’ of this component object.

public Container getParent( ); 

 gets the container in which this component resides. 

public Dimension getSize( );

 returns the current size of this component as a Dimension object, with public members width and height. Replaces the older size( ) method.

public Dimension preferredSize( ) ; 
public Dimension getPreferredSize( );

 a component returns its preferred display size, given its current layout, contents and insets ..etc Used by LayoutManagers to correctly size a component. preferredSize( ) method has been replaced by getPreferredSize( ).

public Dimension getMinimumSize( );
public Dimension getMaximumSize( );

 returns the minimum or maximum dimensions tolerated by this component to display correctly on the screen. 
Used by LayoutManagers to correctly size a component.
These methods replace the older minimumSize( ) method.

public boolean handleEvent(Event e) ;

 a generic event handling method used by Java 1.02
see Java 1.02 Event model.
public boolean action(Event e, Object what );

public boolean gotFocus(Event e) ;
public boolean lostFocus(Event e);
public boolean mouseDown(Event e);
public boolean mouseUp(Event e);
public boolean mouseEnter(Event e);
public boolean mouseExit(Event e);
public boolean mouseDrag(Event e); 		 more specific event handlers which are called by the handleEvent( ) method, depending upon the type of event and the type of component.
see Java 1.02 Event model.
public synchronized void addComponentListener (ComponentListener c);

public synchronized void addActionListener (ActionListener a);

public synchronized void addFocusListener (FocusListener f);

public synchronized void addKeyListener (KeyListener k);

public synchronized void addMouseListener (MouseListener m);		 see Java 1.1 Event model.

public void show( ); 
public void hide( );

 Deprecated for Java 1.1. Can be called to make the component visible / invisible.  

public void setVisible(boolean b);

 this method replaces the show( ) and hide( ) methods. If the argument b is true the component is visible else it does not display.

public boolean isShowing( );
public boolean isVisible( ); 

 tests to see whether the component is currently visible on the screen.

public void invalidate( ) ; 

 marks the Component as needing laying out by the LayoutManager.

public void layout( );
public void doLayout( ); 

 if this component is also a container, this method prompts the LayoutManger to lay out this component. layout( ) has been replaced by doLayout( ) in Java 1.1.

public void repaint() ; 

 requests that at the next opportunity the AWT thread should update( ) this component (see update method below).

public void update(Graphics g) ; 

 the AWT thread calls this method, which by default clears the graphics area of the component to the background color and in turn calls the paint( ) method below, passing the Graphics area.

public void paint(Graphics g) ; 

 the AWT thread calls this method to allow the component to paint itself on the screen.
