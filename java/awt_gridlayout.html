java.awt.GridLayout

 GridBagLayout

A GridLayout arranges components in a rectangular grid like a table of cells, a spreadsheet or calendar. Components occupy cells filling them from top left to bottom right, in the order they are added to the container. There is no way to skip over a cell, but you can add a 'dummy' component (empty panel) to give the appearance of an empty cell.

GridLayout is useful for containers that are laid out in a regular grid, like a calendar or spreadsheet. However it ignores the preferred size of the component, and stretches it to fit the entire cell of the grid!
Buttons, Choices and CheckBoxes look strange when stretched beyond their minimum size, they should not normally be added directly to a GridLayout.
Similarly TextField and Scrollbars should be nested within a Panel with a LayoutManager that limits their heights.

The constructors for GridLayout are as follows :

GridLayout(int rows, int cols);

GridLayout(int rows, int cols, int hgap, int vgap);

Note: When you define a new GridLayout and connect it to a container, you specify the number of rows and columns in the grid, and the horizontal and vertical gaps you want between cells of the grid. However GridLayout makes its own calculation of the number of columns needed to layout all the components in the container (taking into account the container's height, insets, vgap and number of rows).
