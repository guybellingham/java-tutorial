A method is similar to a function() call in other programming languages.
You code methods within the body of the Java class statement. Parameters can be passed to a method, which then does some kind of processing with them, and can optionally return a value, to the caller.
Here is the structure of a method statement.

Each method in a class must have a unique ‘signature’, which is the combination of the method name and the number and types of arguments in the parameter list.
The method name is required and must be followed by parentheses, which may or may not contain, a comma separated list of arguments to be passed into the method code block. When you reference a method, Java uses this 'signature' to look for the correct method block to call at runtime.
Note: The return type and any exceptions thrown by a method play no part in uniquely identifying a method by its signature.
So the following example class would fail to compile. 

package my.pets; 
public class Cat extends Pet {
 //Stroke the cat x times 
 public void stroke(int x) {
 } 
 //Compile Error! - duplicates the method signature above 
 public Purr stroke(int y) throws ScratchException {
 }
}

By convention method names start with a lowercase word, which is usually a verb or action, and can be followed by any number of Capitalized words.

A method has single return type which may be, an object type, a datatype, an array type or the keyword void.
The void keyword is used to define a method that 'does something' without returning a result. It is quite valid to call a method, and ignore the returned object or datatype. This is termed calling a method for its 'side effects', and is why the return type plays no part in finding a method by its signature. 

package my.pets;
public class Pet {
  //The age in years of this Pet - defaults to zero
  int myAge;
  //The PetLicense object for this Pet 
  PetLicense license;
  //Returns my PetLicense object
  public PetLicense getLicense() {
    return license;
  }
  //Returns an integer datatype
  public int getAge() {
    return myAge;
  }
  //Returns nothing
  public void setAge(int newAge) {
    myAge = newAge;
  }
}

Methods are inherited from a parent or super class. When you reference a method, in an object, Java looks for a method with that same 'signature' in the class definition of that object. If none is found, Java searches for that method, upwards through the inheritance hierarchy, first in the 'parent' or super class, and then in the 'grand-parent' and so on.

Inherited methods can be overridden. If the behavior of an inherited method is not what your class needs, you can define your own version of the method with the exact same signature and return type.
So for example, if my Cat class wants to return an age in 'cats years', instead of human years, it can override the getAge( ) method defined in the Pet super class like this.

package my.pets;
public class Cat extends Pet {
  //Add a new method for Cats only 
  public String say() {
    return new String("Meow.");
  }
  //Overrides the method in the Pet class
  //Returns my age in Cats years
  public int getAge() {
    return myAge * 7;
  }
  //Inherits the setAge( ) method from Pet
  
}

There are some rules to remember when overriding a method in a super class.

  • Access to the method in the subclass cannot be made more restrictive (more private), than the method in the super class that is being overridden. However access can be made less restrictive (more public).
  • The overriding method in the subclass may opt to become synchronized (thread safe), when the overridden method is not, and vice versa.
  • If a method is defined as final, by definition it cannot be overridden by a subclass.
  • The method name, parameter list and return type, must match exactly the overridden method in the super class.
  • The subclass can throw the same Exception objects (or subclasses of those), that are thrown by the overridden method in the super class. The subclass may opt not to throw exceptions, but it cannot throw new exception types. Unchecked (runtime) errors and exceptions may be thrown as and when needed.

Methods can be overloaded. If you define more than one method with the same name in a class, you 'overload' that method. Each definition of the method must have different parameter lists, in effect providing several alternative ways to achieve the same behavior. This is often done with special methods called constructors, and gives you several alternate ways to construct an object.
The java.awt.Point class provides 3 different constructor methods : 

package java.awt;
public class Point {  
 //Construct a Point object at coordinates 0,0              
 public Point ( ) { }            
 //Construct a Point object at coordinates x,y              
 public Point (int x, int y) { } 
 //Construct a Point object using an existing Point              
 public Point (Point p) { }
}
