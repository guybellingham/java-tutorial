Object Inheritance.

 organizing classes

Java objects do not exist in isolation. A type of object (like a Circle) is always built upon a 'parent' type, which is called the super class (such as Shape). It inherits all the traits of the parent type. The 'parent' class in turn inherits from the 'grandparent' class, and so on, up the family tree.

At the top of the inheritance hierarchy is an abstract class called java.lang.Object, from which all other Java classes are descended. This means that all Java objects, inherit some common traits, like Javas automatic storage management.

Java classes are organized into hierarchical libraries, much like the folders and files used in file systems. Related classes of objects are packaged together, and these packages can be imported for use in your own Java programs or class definitions. We'll look at the package keyword on the next page.

In the example shown below, I have defined 3 object types, and packaged them into a library called my.pets.

The fully qualified names of these classes of object are :

  • my.pets.Pet - represents all Pets as having a name and age.
  • my.pets.Dog - extends the Pet class, and defines a Dog as having a bark and a bite.
  • my.pets.Cat - extends the Pet class, and adds a cats breed and whiskers.

The full name of a class, uses 'dot notation'. It is the package name, followed by the class name.

The full class name must be unique, and the accepted standard, is to begin your package names with your internet domain name, in reverse. For example my classes should live in packages that start com.spiritone.gus. ...etc
