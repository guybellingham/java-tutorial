The java.io.File class takes pathname and filename arguments in its constructor methods. The File class, is not used to create (write) a new file, for that use a FileOutputStream or RandomAccessFile object. The File class, allows you to work with directories and files in a hardware and file system independent way. You can rename and delete files, create directories, and find out if a file already exists, using methods defined in a File object.
Note: Windows uses the backslash separator '\' in filenames, while Unix uses '/'. The java.io.File class defines a public final static variable called separator, which you can use to construct portable filenames.

The java.io.RandomAccessFile class allows you to both read from and write to a file. By implementing both the DataInput and DataOutput interfaces, and not treating a file as a sequence of bytes only for input or output, the RandomAccessFile class, trancends the notion of streams. You contruct a RandomAccessFile around a file 'path' String and a permission String. The permission must be either "r" for read only, or "rw" for both read and write. A File object may be passed in place of the path String argument to the constructor method. In addition to the methods defined by DataInput and DataOutput, a RandomAccessFile has the following methods to support positioning a pointer within a file.

  • void seek(long newposition); Sets the file's pointer to the new position relative to the start of the file.
  • long length( ); Returns the current length of the file in bytes.
  • long getFilePointer( ); Returns the current RBA (relative byte address) of the file's pointer.

The java.io.StreamTokenizer class makes it easy to extract 'tokens' from a text file. A typical application first constructs an instance of this class, sets up the syntax tables, and then repeatedly loops calling the nextToken method in each iteration of the loop until it returns the value StreamTokenizer.TT_EOF. After a call to the nextToken() method, the ttype field contains the type of the token just read. For a single character token, its value is the single character, converted to an integer. For a quoted string, its value is the quote character. Otherwise, its value is one of the following:

  • TT_WORD indicates that the token is a word.
  • TT_NUMBER indicates that the token is a number.
  • TT_EOL indicates that the end of line has been read. The field can only have this value if the eolIsSignificant method has been called with the argument true.
  • TT_EOF indicates that the end of the input stream has been reached.
