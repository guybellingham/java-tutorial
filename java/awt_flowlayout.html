java.awt.FlowLayout

 BorderLayout

In a container with a FlowLayout manager, components are arranged from left -> right, and top -> down. As new components are added to the container, FlowLayout positions each on the same line as the previous one until they no longer fit (given the width of the container). Then the row is centered and a new row is started below and the next component is added to its' center. FlowLayout, respects a components preferred width and height, even if the component will not fit in the width or height provided. However it may move components from one row to another in the container as it resizes! It is used mostly for, fixed size containers that lay their components out in rows and columns, from top-left, to bottom-right.

The constructors for a FlowLayout are as follows :

FlowLayout( );

Default constructor, aligns components in the center of each row. Horizontal and Vertical gaps are 5 pixels.

FlowLayout(int align);

Pass the constants defined in FlowLayout.LEFT, CENTER, RIGHT to control alignment on a row. Default horizontal and vertical gaps are still 5 pixels.

FlowLayout(int align,
int hgap, int vgap);

 Specify alignment and horizontal and vertical gaps between components. One trick to make components align in a column is to set horizontal gap to a large number that cannot be matched by the container width.

Note: A single FlowLayout manager serves as the default LayoutManager for all instances of the Panel container class.
