What is a Java Object?

 Advantages of Objects

Everything in Java is an object, a Java program is a type of object, and in essence contains a collection of other objects, which send and receive messages telling one another what needs to be done. The complexity of a conventional 'program' is broken up into interactions between self contained packets of Java code and data (objects).

An object exists at runtime in computer memory, and respresents a particular instance of an object definition. Objects have code (also termed behaviors) and data (also known as properties). Objects generally serve a single functional purpose. Java refers to objects, using their address in memory. Objects exist only when a Java program is running. Objects are loaded at runtime, from definition files that have a .class extension. These 'class files' contain all the information needed to construct an object, initialize its data and define the things that it can then do at runtime.

Objects, have a number of essential characteristics, the most important of which are :

  • encapsulation - the ability to hide their data and logic away, and to be self contained and portable.
  • inheritance - the ability to base one object type upon an existing object type, and extend its functionality in some desired way.
  • polymorphism - this is a tricky one to explain, and we'll cover it in more detail on a later page. Essentially it means that you can handle an object as if it were its more generic parent or grandparent object. This is very useful when you want your code to be able to work with a variety of objects that all inherit from a generic parent object type.

This is a common way of depicting an object, with its variables (or 'properties') protected inside a 'shell' that consists of the public methods (the 'behaviors').

Other objects can act on the variables of this object, but only in a controlled way, through the available methods. The ideal is to advertise what an object can do but hide how it does it. As we'll see in a later page, Java has keywords that let you control access to methods and variables inside an object.

 

Objects can contain other objects. So that in addition to being able to derive new object types from existing ones, you can also build new types of objects, by assembling other existing objects, much like building with Lego bricks.
