javax.swing.JComponent

 JLabel

All Swing components are compatible with the Java 1.1 Java Beans architecture. That means basically that they use EventListeners to handle events and they have consistent property API (using get and set methods).

Since JComponent inherits from java.awt.Container, all Swing components can contain other Components for example a JTable can contain JButtons and JLabels etc.

All these user interface controls, seperate the user interface (View and Controller) from the underlying data model. This means that you can customize just the look and feel of a control while reusing the underlying data model. Actually the reverse situation is more normal, where you want to 'extend' the default data model of a control, while reusing the existing presentation logic.

JComponent is an abstract class, its constructor method is only used by derived classes when they are instantiated. Functionality common to all Swing components can be found in the JComponent class. This important class is responsible for managing the pluggable look-and-feel, handling accelerator keystrokes and actions, supporting borders and the sizing of all components.
In addition JComponent adds support for accessibility, internationalization & locales, tooltips, auto-scrolling, focus and double buffering for fast repainting.
