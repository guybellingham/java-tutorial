DataInput and DataOutput

 Readers & Writers

The FilterInputStream and FilterOutputStream classes define abstract methods of filtering incoming or outgoing streams of bytes.

Filter stream classes, always require some other InputStream or OutputStream (as appropriate), to provide the stream of incoming or outgoing bytes.

You can chain together as many stream filters as you need, each one just attaches to the last, and continues adding processing to the stream of incoming or outgoing bytes.

figure: How filter Stream classes attach to Input or OutputStream classes.

DataInput and DataOutput are two interfaces which layout methods for reading and writing the fundamental datatypes.

Dealing with bytes of data is cumbersome, java programs usually need to read or write highly structured data. Another problem with bytes is that the 8-bit encoding schemes used, do not accommodate international character sets.

figure: The Input and Output Filter Stream classes in the java.io package.

Of the many different classes which specialize in filtering streams, the DataInputStream and DataOutputStream are important, because they implement 2 interfaces (shown in red), which declare methods for dealing with the fundamental Java datatypes, short, integer, long, float, double, char, and boolean in addition to plain old byte. PrintStream and RandomAccessFile, also define these methods.

Note: Java 1.1 introduced two new character based (16-bit) Input and Output streams, called Reader and Writer, which are faster at retrieving blocks of data, and make it easier to develop programs in foreign languages.

DataInput and DataOutput define the following abstract methods.

method description
boolean readBoolean( ) throws IOException ; Reads a true/false value
byte readByte( ) throws IOException ; Reads a signed (2’s complement) byte.
int readUnsignedByte( ) throws IOException ; Returns an unsigned byte of data, as an int.
short readShort( ) throws IOException ; Reads a signed 2-byte numeric value. The first byte read is the ‘high-order’ byte.
char readChar( ) throws IOException ; Reads a 2 byte Unicode character.
int readInt( ) throws IOException ; Reads a signed 4 byte integer value.
long readLong( ) throws IOException ; Reads a signed 8 byte long integer value.
float readFloat( ) throws IOException ; Reads a 4 byte representation of a signed decimal number.
double readDouble( ) throws IOException ; Reads an 8 byte representation of a signed decimal number.
String readLine( ) throws IOException ; Reads a sequence of bytes and converts them to a string of Unicode characters, with the high-order 8 bits set to zero. Reading continues until a newline char (‘\n’), carriage return (‘\r’) char or end-of-file condition is read.
void readFully(byte dest[ ]) throws IOException ; Fills the byte array called dest, from the input stream. An EOFException can be thrown if the stream ends before the array is filled.
void skipBytes(int num) throws IOException ; Attempts to read and discard num bytes from the input stream.
void writeBoolean(boolean b) throws IOException ; Writes boolean values, such that (byte)0 represents true, and (byte)1 represents false.
void writeByte(int i) throws IOException ; Writes the high-order byte of integer i.
void writeShort(int i) throws IOException ; Writes the two low-order bytes of integer i (bits 8 – 15 first followed by bits 0 – 7).
void writeInt(int i) throws IOException ; Writes all four bytes of integer i. Starting with the high-order byte (bits 24 – 31).
void writeLong(long l) throws IOException ; Writes eight bytes of long l. Starting with the high-order byte (bits 56 – 63).
void writeFloat(float f) throws IOException ; Writes the 4 byte representation of a floating point number f.
void writeBytes(String s) throws IOException ; Writes the String s, as a series of bytes. Only the low-order byte of each Unicode character is written.
void writeChars(String s) throws IOException ; Writes the String s, as a series of chars. Two bytes are written for each Unicode character.
