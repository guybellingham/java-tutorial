Summary of Basics.

 programming
  • So far, we've seen how to define a type of object, using the class statement. It is important to distinguish a class definition from an actual object. The class acts like a sort of cookie cutter, whereas an object is like an individual cookie made with the cookie cutter.
  • You know that a class definition usually extends (is based upon) an existing class of object. We've talked about how objects inherit the methods (the logic) and variables (the data) of all the super classes that they extend. In fact classes have an inheritance hierarchy (a family tree), and the java.lang.Object class (at the root of the tree) is the grand daddy of all Java classes.
  • Java allows you to package your classes using a folder structure of your own choosing. This helps you to group related classes and ensure that class names are unique and do not clash with existing classes. To uniquely identify a class to the compiler, you may type its fully qualified class name (which is the package folders and subfolders, followed by the class name, all seperated by the full stop character).You may also use the import statement to identify to the compiler, which package folders it should search to find the classes referenced from within your class definition.
  • We've also discussed the public, protected and private keywords, that allow you to encapsulate or hide data and logic inside your objects.
  • Java ClassLoaders are responsible for finding and loading the compiled .class files and building objects by invoking the special constructor method which returns the address of the newly constructed object in memory.
  • You've seen that Java objects are polymorphic; that is, they can be treated as if they were a more generic but related class and can even be handled by any programming interface they implement.
  • To give you more flexibility in designing a class (or type) of object, Java allows you to define an interface. An interface is like a class, except that you do not flesh out the member methods (they are abstract), and any member variables are automatically defined as constant (or final) values. An interface is a sort of contract or protocol of behavior, which can be implemented by classes. Classes that implement an interface are agreeing to override the empty methods in the interface with their own methods.
  • You should now know how to construct and refer to a new object in memory at runtime, using an object variable. You assign the new instance of a class to an object variable or 'handle'. It is through the object variable, that you indirectly reference the methods and variables of the object, using a 'dot notation'. Note: When assigning objects with the = operator, and comparing objects with the == operator you are actually dealing with object addresses and not the objects themselves.
  • Within the class statement you define member methods (the logic) and member variables (the data). Variables defined outside of any code blocks in a class, are called instance variables and are accessible from anywhere within the class. Variables defined within a method code block (known as local variables) are accessible only within the curly braces of that code block.
  • A method statement, defines a single return type (or the keyword void), the method name and optionally any parameters expected by the method in parentheses. The combination of the method name and the parameters it accepts is the methods signature, which must be unique within a class. Methods may be overloaded, having the same name, with different combinations of parameters. Methods in a subclass may override methods inherited from a super class if they have the same signature and return type.
