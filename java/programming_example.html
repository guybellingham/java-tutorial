Writing a Java application.

STEP#1. In this first program, we are going to write an application that has no graphical user interface. GUI programming in java is a complicated subject involving the large and complex AWT and SWING class libraries. This command line application will build upon the subjects covered in the programming section, and assumes you have already read the instructions on downloading the J2SE development kit and run the ExampleProgram to verify the install.

As noted in the downloading instructions, the Java runtime, looks for a method with the signature : public static void main(String[] arg) to start executing an application.
Now you need to understand that this a static method, which means it is not associated with the world of objects, it is 'outside' the realm of individual objects and constructors and inheritance. Instead this method is associated with the class as a whole, and is built when the class file is first loaded into memory. The thing about static methods, is that you are limited to referencing other static methods and variables, you cannot reference an individual object, unless someone else passes you a reference to one.
To program in the object world, we need to do a sort of 'bootstrap' maneuver (pulling yourself up by your own bootstraps) to construct an object based upon the applications class (an instance of ourself) as shown below.

STEP#2. This class definition is the start of our application. 

package examples; 
import java.io.*; 
import java.text.*; 
import java.util.*;  
/** 
 * A command line App, demonstrates Strings. 
 * @author: Guy Bellingham-Smith 
 * @see java.io.BufferedReader 
 * @see java.lang.System 
 */ 
public class Example1 { 
  //class variable 
  private static String s0 = "Point"; 
  //instance variable 
  private String s1 = "Point"; 
  //Character IO Reader 
  BufferedReader buf;
               
  /**
   * Example1 constructor.
   * @exception IOException
   */
  public Example1() throws IOException {
    super();
    //local input stream reader
    InputStreamReader isr = new InputStreamReader(System.in);
    //instance buffered reader wraps isr
    buf = new BufferedReader(isr);
  }
  /**
   *  Entry point of runnable app.
   */
  public static void main(String[] args) throws IOException {
    //bootstrap the app object
    Example1 app = new Example1();
  }
}

Notice that the class name is Example1.
I have included a package statement (as the first statement), after which I used the import statements to make it easy to reference classes in other java packages.
This source code includes some javadoc comments (between /** and */).
The class contains a static String variable called s0, and an instance String variable called s1. It also contains an instance variable of the java.io.BufferedReader class called buf, which is assigned to a new BufferedReader object in the class constructor method.
The class contains a single constructor method (with no return type), that invokes its parent class constructor with the super( ); statement. This enables any parent classes to do their own thing, during object construction and is not strictly necessary, since the java compiler inserts this statement if you forget it. The constructor method goes on to build a java.io.InputStreamReader object called isr. Note that this statement and the following one can both throw a java.io.IOException, but that this method simply passes the buck, by declaring that it can also throw this type of exception up the calling chain.
Lastly a main( ) method is the entry point into our application. The first thing it does is create an object based upon our class. Because this involves calling the constructor method which can throw an IOException, our main method must either catch such exceptions or re-throw them, in this case it simply throws any exception generated back to the java runtime.

STEP#3. You will need to save this class definition with a filename of Example1.java.

Because I have chosen the package name, examples, I must ensure that the executable class file ends up in a folder called \examples with respect to my java home folder. Actually it can be with respect to any folder that I have included in the Java CLASSPATH.

STEP#4. To compile our class you'll need to run the javac tool from the command line.

 

Once the .java source file has successfully compiled, you should see a corresponding .class file in the \examples folder.

A Java application can be run from the command line, by invoking the java executable and passing it the name of the main class.
At the moment our application doesn't do anything very exciting, so if you try running it you won't see any output. If you get a java.lang.NoClassDefFoundError its probably because you're trying to run Example1, you need to specify the fully qualified class name. Try changing directory to your java home folder, and typing
> java examples.Example1

STEP#5. Now we're going to add a method that makes use of the buffered character reader called buf. Copy and paste this method into your Example1.java class definition. 

/** 
* get user input as a String. 
*/ 
public String getUserInput() throws IOException { 
   return buf.readLine(); 
}

The buf object was built in our constructor method, by 'wrapping' a local InputStreamReader object called isr. This is a common technique for adding layers around an object. You'll see that when we constructed the InputStreamReader object, we wrapped yet another object called System.in.

Next copy and paste the statements in red into the main( ) method of your Example1.java class definition. Notice that to reference instance variables or methods of our class, we must use the app object, as in app.getUserInput(); 

public static void main(String[] args)              
  throws IOException { 
  //bootstrap the app object 
  Example1 app = new Example1(); 
  //Prompt the user 
  System.out.print("Please enter a phrase: "); 
  //Get the user input 
  StringBuffer sb = new StringBuffer(app.getUserInput()); 
  //Echo the input backwards 
  System.out.println("Backwards that is " + sb.reverse()); 
}

Note: System.in is an object that the java runtime environment provides for you. It corresponds to the system console (or command line) input stream. We'll talk more about Streams when we look at the Java IO (input and output) classes. There is also a System.out object available to you, and it provides a number of useful print( ) and println( ) methods for printing text to the command line.

STEP#6. Recompile your Example1.java file as shown in STEP#4 above. Run your examples.Example1 application as shown below.

Congratulations!
