AWT Layout Managers

 FlowLayout

Default LayoutManagers.
All the AWT Containers have a default LayoutManager, for the Panel and Applet, it is the FlowLayout. For a Frame it is the BorderLayout.

Nesting Containers and LayoutManagers.
On their own, none of the standard LayoutManagers will be able to cope with the demands of a complex interface. As you design more complex programs, you will need to 'nest' containers and their associated LayoutManagers, inside one another.

INSETS.
The standard LayoutManagers, supplied with the AWT, are careful not to place components, anywhere within a margin around the inside of all Containers. Access to the size and arrangement of this margin is provided by all Containers, with the following method :

public Insets getInsets() { ...return the insets of this Container }

Containers, don't supply a method to change or set the value of their Insets, you must extend a class derived from Container and override this method, with your own, if you want to make use of this margin.

Note: The Frame container, makes use of Insets, that by default include a margin for a MenuBar at the top.
Beware: While the AWT LayoutManagers obey a containers Insets, there is nothing stopping you from drawing anywhere you please within a container's Graphics area, and that might include the area beneath the MenuBar of a Frame!

PREFERRED SIZES.
LayoutManagers are called upon to calculate a container's 'preferred size', with the preferredLayoutSize() method. Typically, the method cycles through every component within the container, and determines the components' own, preferred or minimum sizes.
LayoutManagers do not always honor a components' preferred size, in fact some layout managers like the GridLayout and BorderLayout will ignore the preferred sizes returned by components, and will stretch a component to fill a layout requirement.

For example : a FlowLayout honors the preferred width and height of components, but a GridLayout ignores the preferred size, and stretches the width and height of components to fit the grid. A BorderLayout honors the preferred height of components added to the NORTH and SOUTH, but stretches the width to fill the container. Components placed to the EAST and WEST, keep their preferred width but are stretched to fill the vertical space between north and south components. A component added to the CENTER of a BorderLayout is stretched in both directions to fill any empty space left over.

Beware: The default getPreferredSize() for the Canvas component is 0,0 (width, height), and you will not see a Canvas, if you place it within a container, that has a LayoutManager that respects both horizontal and vertical preferredSizes! You can get around this, by extending Canvas with your own class, and overriding the getPreferredSize() method, to give more sensible defaults. Also see the Frame container.
