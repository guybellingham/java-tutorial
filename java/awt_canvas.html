java.awt.Canvas

 Checkbox
Like all awt Components, a Canvas has its own Graphics object.
Use the getGraphics( ) method to get a handle on a Component's Graphics context.
You can control the drawing color with the setColor(Color c) method of the Graphics class.
The Applet at left contains a subclass of Canvas, that has a thread that continuously draws Ripple objects wherever you click the mouse.

Try clicking while a Ripple is still drawing. The Canvas subclass can handle more than one Ripple at a time.

Note: If you scroll the Applet offscreen, the drawing on the Canvas is lost whenever the canvas component has to be redrawn by the AWT thread that handles painting and events.
That is because the Ripples are not properties of the canvas object, they are just drawn into the Graphics context of the canvas and then 'forgotten', the Applet 'remembers' nothing about the drawing you do, and so it is lost, whenever the screen area needs repainting.
