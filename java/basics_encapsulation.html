Object Encapsulation.

 member methods

Java helps you to encapsulate your objects. This means you can hide the inner workings (the implementation) and only expose the public interface you want other objects to see.

  • This creates a clear understanding of how your object is to be used (through the public interface).
  • It prevents other objects from delving into the workings of your object and messing things up (avoiding nasty side effects).
  • It allows you to change the way your object works while maintaining the same interface (avoiding a ripple effect of changes).

Java uses the keywords public, protected and private to declare different levels of access to the variables and methods inside a class of object. We'll look at member methods and variables on the next pages.

You can also use the public keyword on your object definition, to signify that it is generally accessible to other types of objects, in other packages (in other libraries). For example most of the built in Java classes are defined as public, and allow you to import them, and create objects from them and refer to their methods and variables from within your own classes and packages.

These keywords, control access to variables and methods in the following general way (going from least to most restricted) :

  • public - a public variable or method is accessible from any class that can access the class that contains that member variable or method.
  • protected - a protected member variable or method can be accessed from any class within the same package and by classes defined in other packages that inherit from the members class.
  • private - You can only reference a private member from within the class in which it is defined.
Diagram, showing access to member variables and methods across packages.

This diagram shows the effect that putting classes in different packages has upon access rights.

Notice that the Cat class is not defined as public. This means that classes in other packages cannot even 'see' the Cat class.

The Dog class is public, and classes in other packages do have access to its public members (all classes can reference rovers name). However, the yellow arrow means that only classes in other packages that inherit from Dog, can access protected members, like rovers bark.

Members with no access modifier, and those that are declared private are not accessible to classes in other packages.

Diagram, showing access to member variables and methods within a package.

Classes within the same package, have much less restrictions on access to one anothers member variables and methods.
