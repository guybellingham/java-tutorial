java.awt.LayoutManager

 Layout

Every AWT Container has one LayoutManager, which is responsible for arranging the components within the container.
Whenever the visual aspect of a Container changes (on first display, or when the container is resized), the AWT invokes the container's validate( ) method, which in turn calls the layout( ) method. This method relies upon the layoutContainer( ) method in the layout manager to position all the components in the container.

Java defines the general properties and behaviors common to all layout managers in the java.awt.LayoutManager and LayoutManager2 interfaces, with methods for adding and removing components and laying out the contents of a container.
Below are shown the five built-in layout managers that are included with Java 1.0 and which implement these interfaces.

All LayoutManagers, should define the following methods :

void addLayoutComponent(String name, Component comp);
void removeLayoutComponent(String name, Component comp);
Dimension preferredLayoutSize(Container parent);
Dimension minimumLayoutSize(Container parent);
void layoutContainer(Container parent);

Note: Although Layout Managers, define these methods, they do not necessarily have to act upon all these methods. For instance, the FlowLayout does not need to 'do' anything in the addLayoutComponent() or removeLayoutComponent() methods, because it simply lays out whatever components it finds in the container that calls it (a container maintains an array of components), in the order they were added to that container. Other more sophisticated layout managers, such as BorderLayout, do maintain their own list of components with associated positioning information. When adding a component to a container with a BorderLayout you can specify a String in the add( ) method that describes the desired position of that component.

A LayoutManager is responsible for finding out the 'preferred' and minimum sizes of components, and laying out those components within their container. Some types of LayoutManager work for one and only one container, other LayoutManagers might end up working for any number of Containers, and must be passed a reference to the 'owning' Container, so that they know which one to go to work on.

Generally you use a combination of layout managers and containers in an application to get the desired look.

Java's default Container methods invoke the appropriate methods in the LayoutManager for you, as follows :

Container method LayoutManager method
add( ) addLayoutComponent( )
layout( ) layoutContainer( )
minimumSize( ) minimumLayoutSize( )
preferredSize( ) preferredLayoutSize( )
remove( ) removeLayoutComponent( )

You connect a layout manager to a container using the setLayout(LayoutManager) method : 

setLayout(new BorderLayout( ));

Note: You do not normally need to tell the LayoutManager to recalculate the best layout for the container, the AWT automatically takes care of this for you. However you can 'mark' a container as needing laying out with a call to the container's invalidate( ) method.
