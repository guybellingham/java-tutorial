Member variables.

 creating objects

You can declare variables within a class or method block. Variables must be defined to be of a particular type. You declare variables to reserve memory for some data (text, numbers, arrays...etc) or to reference another object.
Variables must have a name so that you can refer to them, and you should give them an initial value. For example : 

package my.plants; 
public class Tree extends Plant {
  //The number of sap rings for this tree  
  int rings = 25;
  public boolean isReadyToCut( ) {
    //Whether this tree is ready for harvest true/false
    boolean b = false;
    if (rings > 30) {
      b = true;
    }
    return b;
  }
}

There are 3 basic uses for variables :

  • Variables can be used to declare one of the data types built into the Java language. 
    int yourAge = 0;
    float taxRate = 0.28F; 
    char newLine = '\n'; 
    boolean myFlag = false;
  • You can use an object variable to refer to any class of object such as a Cat or a Circle object. By default an object reference is null , until you assign it to the correct type of object.

    Color myColor; //Beware is null initially!
    Font myBigFont = new Font("Helvetica",Font.BOLD,36);
    Cat puss = new Cat("puss",12);
  • Variables can also be used to refer to an Array of datatypes or an Array of a class of object.

    int [] monthsStartingWithJ = new int[3];
    Shape [] shapeList = new Shape[64];

Variables have different scopes, depending upon where in a class they are defined:

  • Class variables, do not depend upon the existence of any one object. Class variables are defined within the body of the class statement (outside of any member method) using the static keyword. Class variables should be referenced using the class name. They are 'shared' across all instances of the class in which they are declared. For example the java.lang.Math class has a class variable called PI declared like this : 
    public final class Math {
 /** 
      * The double value that is closer than any other to 
  * pi, the ratio of the circumference of a circle to its diameter.              
  */     
 public static final double PI = 3.14159265358979323846;

}

    The keyword static, can be used to declare both variables and methods as belonging to a class as a whole, instead of new objects built from that class. The keyword final makes the value of PI a constant. To get the value of PI in your classes, you can reference this class constant like this :

    double circumference = 2 * Math.PI * radius;

    Notice that the value PI is not dependent upon the existence of a Math object, we did not have to construct a Math object to reference PI, we just used the class name.

  • Instance or object variables, belong to a particular object, (their existence is tied to a single runtime instance of a class) and they have a lesser scope. They can only be referenced through an object variable. They are usually declared just after the class statement (outside of any member method). The Java compiler does not force you to assign an initial value. Java automatically initializes numeric variables to zero, character variables to unicode '\u0000' and booleans to false. Uninitialized object variables are null. For example the java.lang.String class, represents a sequence (or string) of character text, it defines a private instance variable called count, which is defined thus : 
    public final class String implements java.io.Serializable, Comparable {
 /** The count is the number of characters in the String. */
 private int count;
 /** Returns the length of this string.
  * The length is equal to the number of 16-bit
  * Unicode characters in the string.
  * @return  the length of the sequence of characters represented by this
  *          object.
  */
 public int length() {
  return count;
 }
}

    Obviously the count of characters in a String, depends upon the particular text in a String object, and so it makes sense to have a different count variable, for each instance of a String. Therefore you want to call the length() method only on a particular String object :

    String s1 = new String("Java");
String s2 = new String("");
    s1.length(); //returns integer 4
s2.length(); //returns integer 0
  • Local or method variables, are defined within a method or code block (within curly braces). The compiler ensures that they must be given an initial value when declared. They exist only for as long as the method or code block is executing in memory. Take a look at the compareTo( ) method in the java.lang.String class, there's some stuff here you may not understand, but the method makes good use of local variables.

There are a couple of rules to remember about defining variables :

  • By convention member variables have meaningful names, and the first word is in lowercase. Names cannot begin with a number or contain spaces.
    Note: Names can begin with the special characters $ and _ (underscore), but most special characters have other uses in Java, so it is better to use meaningful alphabetic names.
  • Names are case sensitive !
  • You cannot define 2 variables with the same name, in the same code block.
  • Local variables must be declared before they can be used.
  • Local variables, must be initialized when they are first declared. If the initial values are not available at the point you need to declare a local variable, you can use 0 for integer datatypes, 0.0F for float and 0.0 for double datatypes, '\u0000' for the char datatype and null for object variables.
  • If you declare a local variable with the same name as an instance variable, the local variable (while it exists) will 'mask' the instance variable. To use a masked instance variable you can refer to it using the special Java keyword this as shown below. Avoid naming clashes, they make your code hard to understand. 
    public class Rectangle {
  //Instance variable called x
  private int x = 0;
  //Set the instance variable x using local variable x
  public void setX(int x) {
    this.x = x;
  }             
}
  • Avoid using Java keywords in your variable names.
