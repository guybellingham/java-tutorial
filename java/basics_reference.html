Working with methods and variables.

 summary

Now we need to look at what happens when you reference the member methods and variables within an object at runtime.

To reference a member variable in another object. Use the object variable that addresses the other object, followed by a fullstop and the name of the instance variable. You cannot directly reference local variables, defined inside method blocks. Remember that generally you can only reference variables in other classes and packages that are defined as public.

A static variable belongs to the class as a whole. If you define a variable with the static keyword, you should use the class name instead of an object variable, to reference that variable.
For example the java.lang.System class has a static variable called out that Java applications can use to print standard system console messages : 

/**
 * The "standard" output stream. This stream is already 
 * open and ready to accept output data. Typically this stream 
 * corresponds to display output or another output destination 
 * specified by the host environment or user. 
 * For simple stand-alone Java applications, a typical way to write
 * a line of output data is:
 * System.out.println(data);
 * @see java.io.PrintStream#println(java.lang.String) 
 */ 
 public final static PrintStream out = nullPrintStream();
To reference a member method in another object. Use the object variable that addresses the other object, followed by a fullstop and the method name followed by the parameters in parentheses. Remember that java finds methods by their 'signature'. 
//DepositAccount extends the generic Account class 
Account account = new DepositAccount();
//Reference the method with the 'deposit(double)' signature
account.deposit(512.50);

The parameters you pass to the method (in parentheses), must match the ‘signature’ of a method definition in the object class (or in the classes it inherits from). If the java compiler can't find the method in the object variable class (or its inheritance hierarchy) you will get a compile time error, saying the method is not defined.

Gotcha! When you reference an object by a more generic parent class, you only have access to the member methods and variables of that parent class. So to get the above example to compile, the Account class must have a definition for the deposit(double amt) method, which the DepositAccount class should override : 

public class Account {
  //All Accounts have some sort of deposit() method 
  void deposit(double amt) { 
    //credit the account here 
  }
}

A static method or variable belongs to the class as a whole. If you define a method with the static keyword, you should use the class name instead of an object variable, to reference that variable or method.
For example the java.lang.Integer class has a static method that can turn a String into an integer value like this : 

public static int parseInt(String s) throws NumberFormatException {
}

Don't worry about the 'throws NumberFormatException' statement for now, we'll cover Exceptions in the programming section. This method takes a String s as a parameter and returns an int (integer datatype). You can reference this method through the class name (without creating a new Integer object) like this : 

String s = "42";
int x = Integer.parseInt( s );

The fundamental datatypes are passed ‘by value’ to a method.
Data variables differ from object variables in that they reserve memory for data values, as opposed to storing a reference to an object in memory. The term 'by value', means that a copy of the original data variable is automatically made by Java and passed to the target method as a parameter. Any changes are made to that copy and not the original variable. When the method returns, the data variable’s value remains unchanged!

The example at left shows a Java application called BankApp, whose main() method, accepts an account number as a command line argument.
It converts the command line argument from a String object into an integer data variable called nbr.
Then it passes the integer variable and another numeric data variable called bal, to a method called getAccount().
The called method, gets copies of the values of the two data variables. It uses the account nbr value to get an Account object.
It sets the bal variable to the balance of the Account object, and returns a String which is the account name.
Now the problem is that the main method, gets the account name value returned, but the bal data variable remains unchanged.

Objects are passed ‘by reference’ to a method. The term 'by reference' means that a 'pointer' to the objects location in memory is passed into the method. To be more accurate, a copy of the object variable (automatically made by the JVM) is passed to the target method. This has a couple of consequences :

  1. Any changes made through the (copied) object variable in the target method, impacts the original object in memory, and remain after the method returns ! In the example, the juggle() method can make the red ball bigger.
  2. However any changes made to the value of the object variable itself inside the target method block, are only made to the copy, and are 'lost' as soon as the method returns.
    In the example, the main() method doesn't see any change to the ball object variable.
