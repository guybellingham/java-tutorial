The java.io package

Filters

The java.io package contains classes related to the reading, writing, and filtering of ‘streams’ of data. A ‘stream’ is a sequence of bytes of data, travelling from some source, to a destination.
The package also contains some non-stream classes like, File, RandomAccessFile and StreamTokenizer.

figure: InputStream and OutputStream are the abstract super classes for handling ‘streams’ of bytes

At the top of the heirarchy of input / output classes, are 2 abstract classes InputStream and OutputStream, that prescribe fundamental io methods like read( ), write( ) and close( ). There is no open() method because creating the stream object automatically opens the stream.
The Filter streams work with other streams, and require an instance of an InputStream or OutputStream (as appropriate) in their constructor methods. By chaining a number of filter streams to an InputStream or OutputStream that is associated with an underlying file or data resource, you can add valuable processing to a stream of bytes.

InputStream defines the following abstract methods.
Many of these methods can run into problems when getting or putting data and throw java.io.IOException.

method description
int read ( ) throws IOException ; Reads one byte at a time. The byte is 'promoted' to an int when returned.
int read (byte dest[ ] ) throws IOException ; Read bytes from the data source, into the dest[ ] byte array object. Returns when the array is full or when no more data is available. The int returned contains the number of bytes read.
int read (byte dest[ ], int offset, int length) throws IOException ;  Attempts to reads length bytes from the data source, into the dest[ ] byte array, starting at offset. Returns –1 when the source has no more data.
void close ( ) throws IOException ;  Releases system resources tied to the data source.
int available( ) throws IOException ;  Returns the number of bytes that are ready to be read in without blocking. Not supported under all operating systems !
long skip (long nbytes) throws IOException ;  Attempts to skip over and discard the next nbytes from the data source. The long returned is the actual number of bytes skipped.
boolean markSupported ( ) ;  Returns true if the input stream supports the mark & reset methods, discussed below.
void mark (int readLimit) ;  Marks the input stream at this point, so that a future reset ( ) call can return, and start re-reading bytes.If more than readLimit bytes are read before the next reset ( ) call, the mark is lost.
void reset ( ) throws IOException ;  Rewinds the input stream to the previously marked position, so that subsequent reads repeat the bytes read since the last mark( ) call.

OutputStream defines the following abstract methods.

method description
void write (int data) throws IOException ; Writes the low-order 8-bits of the argument, ignoring the high-order 24-bits.
void write (byte data[ ]) throws IOException ; Writes an array of bytes to the output destination.
Note: A String can be converted to a byte array with the getBytes( ) method. The high-order byte of each Unicode character will be lost.
void write (byte data[ ], int offset, int length) throws IOException ; Writes a subset of the byte array, beginning at offset and for length bytes.
void flush( ) throws IOException ; Forces any bytes that may have been buffered, out to the stream. Do this before closing a stream.
void close( ) throws IOException ; Releases system resources tied to the data consumer.

Several classes inherit from these two abstract classes, and actually implement these io methods :

Class constructor description
FileInputStream (String path) throws FileNotFoundException; Attempts to open the file for input, and throws FileNotFoundException. A File object may also be passed as an argument to the constructor method.
FileOutputStream (String path) throws IOException;
FileOutputStream (String path, boolean append) throws IOException;		 Tries to open a stream to the file in the path, a File object can be passed as an argument. If append is true, bytes will be added to the end of any existing data in the file.
ByteArrayInputStream ( byte buf[ ] );
ByteArrayInputStream ( byte buf[ ] , int offset, int length );		 Takes its input from an array or a part of an array of bytes. The OutputStream version, writes to an array of bytes which automatically grows to fit the data. An array of bytes can be converted to a String, using the buf [ ].toString( ) method.
StringBufferInputStream(String s); Reads bytes from a String and not a StringBuffer object. Strings cannot be changed, therefore the input is fixed.
PipedInputStream( );
PipedOutputStream( );
PipedInputStream(PipedOutputStream o);
PipedOutputStream(PipedInputStream i);		 The first 2 constructors just creates pipes, which are useless until they are connected (using the connect( ) method). These classes are always used in pairs: piped input is read from a corresponding piped output stream, and piped output must connect to a corresponding piped input stream.
