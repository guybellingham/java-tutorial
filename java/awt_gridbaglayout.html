java.awt.GridBagLayout

 CardLayout

This is the most complex of the LayoutManagers, and offers the most control. It arranges components in a table of cells, using a complex set of rules that are packaged as an associated object of the class java.awt.GridBagConstraints. The constraints, determine how the components fill and align within cells and whether the preferred size of the component in a cell, is respected.

When a component is added to the container, you construct the GridBagConstraints object, and set the instance variables of the GridBagConstraints object to specify :

  • The size and location within the container of the components' cell, width and height may span multiple cells (variables gridx, gridy, gridwidth and gridheight).
  • The size and location within that 'virtual' cell of the component (fill and anchor).
  • What happens to the cell when the container is resized (weightx and weighty).

You associate the constraints with the component when you add it to a container. This method takes a 'clone' of the constraints, so you are free to change and associate the GridBagConstraints object with another component. You can also use the following method of the GridBagLayout manager.

myGridBagLayout.setConstraints(myButton, myGridBagConstraints);

The fill instance variable, determines whether the component's width and height will take their preferred values or completely fill the cell. You can use the constants defined within GridBagConstraints to set this value :
Fill Width Height
NONE (default) preferred preferred
HORIZONTAL Cell width preferred
VERTICAL preferred Cell height
BOTH Cell width Cell height
Anchor Component placement
CENTER (default) center of the cell
NORTHWEST, NORTHEAST, SOUTHWEST, SOUTHEAST corners of the cell
NORTH, SOUTH, EAST, WEST center of the cells edges

This applet, allows you to change the constraints of 4 buttons in a GridBagLayout.

Try changing the gridx and gridy values to make all four buttons fit together in a square.

Then try changing the gridwidth and fill values to line up buttons 1, 2 and 3 above button 4 which should span all threee buttons in the line above.

Note: The align value will only make a noticeable difference when fill is not equal to BOTH.
