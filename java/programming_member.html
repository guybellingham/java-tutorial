Method and variable keywords.

 data types

Java permits the following keywords to precede statements that define variables and methods in a class.
These are in addition to the keywords public protected and private, that allow you to encapsulate data and logic in your classes. When declaring member methods in your class, you have to decide whether you want to define just the signature of the method, or provide an implementation for the method as well. When defining member variables and methods, you have the option to finalize the data or logic so that no subclass can override the data value or method implementation. Methods and variable values can be given a class wide scope or can be specific to particular instances of a class (belong to an object).

An abstract method can be defined only within an abstract class.
An abstract method is one without a body of code, and which is meant to be overridden by subclasses. If a class contains an abstract method, it must also be declared abstract, because it is 'missing' implementation details, and should not be directly instantiated. (The term instantiation, refers to the process of constructing a new object from a class at runtime).

A final variable has a constant value that cannot be overridden by any subclasses. When variables are declared final, the Java compiler knows that their values will not change. So for example instead of a reference to the member variable such as java.lang.Math.PI the compiler might 'optimize' your code and substitute the constant value (3.14159) instead. You must give a final variable an initial value when you declare it.
Gotcha! When you make an object variable final, it means that variable always refers to the same object in memory, the one created by its initializer. However, a method can still reference that object and its public members and even change its properties.

A final method has guaranteed processing that cannot be overridden by any subclasses. A final method, can give you the security, that its processing won’t be overridden by any subclasses. For example the java.lang.String class defines final methods, so that other classes can't subvert or pretend to be Java String objects.
When a method that is declared as final is referenced, the compiler is free to replace the complicated and slow method lookup process, with the ‘inline’ code of that method.

A static variable is associated with the enclosing class, and not with any particular object constructed from the class. This means that the variable exists just once in memory when the class is loaded by the Java runtime. The variable can be referenced using the class name and is 'shared' by all objects. You have to be aware of threading and 'side effects' issues when using static variables.

A static method is not associated with a particular instance of a class. The method implementation is loaded once into memory, when the class is first referenced. This has a couple of important consequences:

  1. You cannot use the keyword this (to refer to the current object)
  2. You cannot refer to a non-static variable or method by name from within the static method. 
//class designed for all Shapes to share
public abstract class Shape {
  //incomplete or generic implementation here  
}

The synchronized keyword can precede a method statement or code block, it causes threads of execution to queue up when entering blocks of code, so that the whole section of code is finished by one thread, before another thread can begin processing it.
Java supports multi-threading, which is a way of doing several things at once with the same processor or multiple CPUs. However certain ‘critical sections’ of code, are vulnerable to errors if more than one thread, is processing that code at the same time.
We will explore the concepts of Java threads and how to control them in a following page.

Invalid combinations. Certain combinations of member keywords do not make sense and are not allowed by the Java compiler.

  • An abstract method cannot also be final. A final method must be fully defined, since no subclasses can override the method and make up for any ‘missing’ or abstract parts of the method implementation.
  • Less obviously a static method cannot also be abstract, since there is no object instance associated with a static method, and therefore no this or super references can be made to the current or parent object, there is no way to override the ‘missing’ body of a static method. A static method is not associated with any object at runtime, and therefore it cannot be overridden by an instance of a subclass.
  • An abstract method cannot also be marked as native. A native method is one written in a platform dependent language (not Java).
